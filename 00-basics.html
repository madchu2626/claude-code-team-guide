Glossary

Here's what all the terms mean — in plain English.

TermWhat it means
Claude The AI. Same one as the website (claude.ai). It can answer questions, write things, analyze data, etc.
Claude Desktop App The app you install on your Mac. It has three tabs: Chat (regular conversation), Code (Claude Code), and Cowork (big tasks). Think of it as your home base.
Claude Code A version of Claude that can see and edit files on your computer, run commands, and take actions — not just chat. It's like giving Claude hands. You access it through the Code tab in the desktop app, or through Terminal.
Claude in Chrome A Chrome browser extension that lets Claude control your browser — click buttons, fill forms, read web pages, and take actions on websites you're logged into. Great for automating repetitive web tasks. See the full section below.
Terminal A text-based app built into your Mac. You type commands and the computer does things. You only need it once (for install). After that, the desktop app handles everything.
Connectors / MCP MCP stands for "Model Context Protocol" — but all you need to know is: connectors let Claude talk to other apps (Slack, Gmail, Google Drive, etc.). You set them up in Settings and then Claude can read/send Slack messages, emails, etc. Think of them as plugins.
Cowork A mode in the desktop app for bigger, multi-step tasks. Instead of chatting back and forth, Claude plans the work, executes it, and delivers finished files. Good for spreadsheets, reports, presentations, research.
CLAUDE.md A text file you can put in a project folder with permanent instructions for Claude. Claude reads it automatically every time. Like a "sticky note" that says "always do X" or "never change Y."
GitHub Actions A free service from GitHub that runs tasks on a schedule (daily, hourly, etc.) on GitHub's servers — even when your computer is off. This is how scheduled workflows work.
Cron Job (cron-job.org) A free website (cron-job.org) that automatically visits a URL on a schedule you set — every minute, every hour, daily, etc. It runs on their servers, so it works even when your computer is off. An alternative to GitHub Actions for running scheduled tasks.

There are several ways to use Claude. Here's a quick overview so you know which is which.

How Does Claude Access My Files?

When you use the Code tab or Cowork, Claude can see the files and folders on your computer — but only the ones you point it to. It works like this:

  • You open the Code tab and navigate to a folder (e.g., your project on the Desktop)
  • Claude can read, create, and edit files inside that folder
  • Claude can also run commands (like scripts or installs) on your computer
  • It does NOT have access to your entire computer by default — just the folder you're working in
  • Everything runs locally on your Mac — your files are not uploaded to the internet

How Do Connectors (MCP) Work?

Connectors are how Claude talks to other apps. Here's the simple version:

  • You go to Settings → Connectors and turn on the ones you want (Slack, Gmail, etc.)
  • Each connector asks you to log in once (click "Allow" in your browser)
  • After that, Claude can use that app — send Slack messages, read emails, etc.
  • Claude can only access what your account has access to (e.g., Slack channels you're in)
  • You can turn off any connector at any time in Settings

Claude in Chrome

Claude in Chrome is a browser extension that lets Claude see and interact with websites in your Chrome browser. Think of it as giving Claude eyes and hands for the web.

What can Claude in Chrome do?

  • Read web pages — Claude can see what's on any page you have open
  • Fill out forms — tell Claude what to enter and it fills in the fields
  • Click buttons and links — Claude can navigate websites for you
  • Use sites you're logged into — since it uses your browser, it has access to any site where you're already signed in (Google Docs, Notion, internal tools, etc.)
  • Extract data from pages — pull information from websites and save it
  • Automate repetitive tasks — like data entry across multiple pages

How to set it up:

  1. Open Google Chrome (or Microsoft Edge)
  2. Go to the Claude in Chrome extension page in the Chrome Web Store
  3. Click "Add to Chrome"
  4. Click "Add extension" in the popup
  5. You'll see the Claude icon appear in your browser toolbar — you're done!

How to use it:

Once the extension is installed, you can use it from the Code tab in the Claude desktop app. Just tell Claude what you want it to do in the browser:

Open google.com and search for "quarterly revenue report template"
Go to my Google Sheet and add a new row with today's date and these numbers: 150, 230, 85
Look at the page I have open in Chrome and summarize what it says
Good to know:
  • Claude in Chrome uses your existing browser logins — no need to re-authenticate anywhere
  • It works with Chrome and Microsoft Edge (not Safari or Firefox)
  • You need a paid Claude plan (Pro or Max) to use it
  • If Claude hits a login page or CAPTCHA, it will pause and ask you to handle it
When should I use Claude in Chrome vs. the desktop app?

Use Claude in Chrome when you need Claude to interact with websites — filling forms, reading web pages, automating browser tasks.

Use the desktop app (Code/Cowork tabs) when you're working with files on your computer — editing documents, running scripts, building reports.
