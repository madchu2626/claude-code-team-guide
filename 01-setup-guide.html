Claude Code — Setup Guide

Read this first: This guide walks you through everything step by step. Follow each step exactly. Don't skip ahead. If something goes wrong, check the "Troubleshooting" box after each section.

Before You Start

You need:

  • A Mac computer (macOS 13 or newer)
  • Wi-Fi / internet connection
  • A Claude account (Pro or Max plan) — ask your manager if you don't have one

STEP 1: Download the Claude Desktop App

The Claude desktop app is the easiest way to use Claude on your Mac. It also has a built-in code window where you can run Claude Code — no Terminal needed for most things.

  1. Go to claude.ai/download in your web browser
  2. Click Download for Mac
  3. Open the downloaded file (it will be in your Downloads folder)
  4. Drag the Claude icon into your Applications folder
  5. Open Claude from your Applications folder (or press Command+Space → type Claude → click it)
  6. Log in with your Claude account (email + password)
Tip — The Code Window: Inside the Claude desktop app, you can open a code window to run Claude Code directly. This is often easier than using Terminal. Look for the terminal/code icon in the app, or use the keyboard shortcut. You can type commands and talk to Claude Code right inside the app — no need to open a separate Terminal window.
Troubleshooting

"Claude can't be opened because it is from an unidentified developer" → Go to System Settings → Privacy & Security → scroll down and click "Open Anyway".

Don't see the download? → Check your Downloads folder. The file is called something like Claude.dmg.

STEP 2: Install Claude Code (Command Line)

Even though the desktop app works great, installing Claude Code in Terminal gives you extra power — like running scheduled workflows automatically. You'll need both.

First, open Terminal:

Terminal is an app already on your Mac. You don't need to install it. (You can also use the code window inside the Claude desktop app — but for this step, let's use Terminal.)

  1. Press Command + Space on your keyboard (this opens Spotlight search)
  2. Type: Terminal
  3. Click on Terminal in the results
  4. A window with a black or white background will appear with a blinking cursor
What is Terminal?
It's like a text-based way to talk to your computer. You type commands, press Enter, and the computer does things. You don't need to understand it deeply — just copy-paste what this guide tells you.

STEP 3: Install Claude Code

  1. Click on the Terminal window so it's active
  2. Copy this entire line (triple-click to select it all, then Command+C):
curl -fsSL https://claude.ai/install.sh | bash
  1. Paste it into Terminal (Command+V)
  2. Press Enter
  3. Wait — you'll see text scrolling. This is normal. It takes about 1-2 minutes.
  4. When it's done, you'll see something like Claude Code installed successfully
Troubleshooting

"Permission denied" → Try this instead:
sudo curl -fsSL https://claude.ai/install.sh | bash
It will ask for your Mac password. Type it (you won't see the letters — that's normal) and press Enter.

"command not found: curl" → You may need to install Xcode tools first. Run:
xcode-select --install
Click "Install" on the popup, wait for it to finish, then try the install command again.

Nothing happens / it hangs → Check your internet connection. Close Terminal, reopen it, and try again.

STEP 4: Restart Terminal

This is important! Claude Code won't work until you do this.

  1. Close the Terminal window (click the red X button, or press Command+Q)
  2. Open Terminal again (Command+Space → type Terminal → click it)

STEP 5: Check That It Worked

  1. In Terminal, copy and paste this:
claude --version
  1. Press Enter
  2. You should see a version number like 1.x.x

If you see a version number → It worked! Move to Step 6.

Troubleshooting

"command not found: claude" → You need to restart Terminal (Step 4). If you already did, try:
source ~/.zshrc
Then try claude --version again.

Still not working? → Try reinstalling (go back to Step 3).

STEP 6: Log In to Claude Code

  1. In Terminal, copy and paste this:
claude auth login
  1. Press Enter
  2. Your web browser will open automatically
  3. Log in with your Claude account (email + password)
  4. Click "Allow" or "Authorize" when asked
  5. Go back to Terminal — you should see a success message
Troubleshooting

Browser didn't open? → Look for a URL in the Terminal output. Copy it and paste it into your browser manually.

"Account not authorized" → Make sure you have a Claude Pro or Max subscription, or that your team admin has added you.

STEP 7: Test It!

Let's make sure everything works. You can do this in Terminal or in the Claude desktop app's code window — both work the same way.

Option A: Using the Claude Desktop App (Recommended)

  1. Open the Claude app
  2. Look for the code window or terminal icon (it may say "Claude Code" or show a >_ icon)
  3. Click it to open the code window
  4. Type something like: What is 2 + 2?
  5. Claude will respond. You should see an answer.

Option B: Using Terminal

  1. In Terminal, type:
claude
  1. Press Enter
  2. Type something like: What is 2 + 2?
  3. Press Enter
  4. Claude will respond. You should see an answer.

If you got a response → Congratulations, you're all set!

To exit the conversation:

  • Press Escape then type /exit and press Enter
  • Or press Ctrl+C twice
Going forward, use whichever you prefer! The Claude desktop app's code window and Terminal do the same thing. Most people find the desktop app easier since it's already open.

STEP 8: Run a Health Check

One last thing — let's make sure everything is healthy:

  1. In Terminal, copy and paste:
claude doctor
  1. Press Enter
  2. You'll see a checklist. Everything should have a green checkmark
Troubleshooting

Red X on something? → Take a screenshot and send it to your team lead. It will show what needs fixing.

STEP 9: Skip Permission Prompts (Important!)

By default, Claude Code will ask you "Allow this action?" over and over before doing anything — editing a file, running a command, etc. This gets annoying fast. Let's fix that.

How to turn off permission prompts:

  1. Open Claude Code Settings:
    • In the Claude desktop app: go to Settings (gear icon or Claude → Settings in the menu bar)
    • In Terminal: type /permissions inside a Claude Code session
  2. Find the permissions or tool approval setting
  3. Set it to "Allow All" (also called "yolo mode")
  4. Save / confirm

That's it. Claude will now read, write, and run commands without asking you each time.

For automated / scheduled workflows:

If you're setting up cron jobs or GitHub Actions that run Claude Code automatically (without a human watching), add this flag:

claude --dangerously-skip-permissions

This is required so the workflow doesn't get stuck waiting for someone to click "Allow."

Is this safe?
Yes, for your use case. This just means Claude won't pause to ask "can I edit this file?" every time. It will still follow the instructions you give it. If you're ever worried, you can always switch back to a stricter mode in Settings.

You're Done!

That's it. Claude Code is installed and ready to use.

What to do next:

  • Keep the Cheat Sheet (separate doc) nearby for quick reference
  • Your scheduled workflows are already set up — you don't need to do anything for those
  • If something breaks, run claude doctor and share the output with your team lead

FAQ

Q: Do I need to keep Terminal open all the time?
A: No. Open it when you need Claude, close it when you're done.

Q: Will this slow down my computer?
A: No. It only uses resources while you're actively using it.

Q: Do I need to update it?
A: It updates automatically. But if something seems off, run claude update.

Q: What if I close my laptop in the middle of a conversation?
A: That's fine. The conversation ends. Start a new one when you open Terminal again.

Q: Is this the same Claude as the website?
A: Yes, same AI. But Claude Code can read and edit files on your computer, run commands, and interact with code — the website version can't do that.

Q: How much does it cost?
A: It's included with your Claude Pro/Max subscription. If your team uses API keys, your team lead manages the budget.
