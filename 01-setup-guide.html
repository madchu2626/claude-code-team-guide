Claude Code — Setup Guide

Read this first: This guide walks you through everything step by step. Follow each step exactly. Don't skip ahead. If something goes wrong, check the "Troubleshooting" box after each section.

Before You Start

You need:

  • A Mac computer (macOS 13 or newer)
  • Wi-Fi / internet connection
  • A Claude account (Pro or Max plan) — ask your manager if you don't have one

STEP 1: Download the Claude Desktop App

The Claude desktop app is the main way you'll use Claude. It has a Code tab for Claude Code and a Cowork tab for bigger tasks — all without needing Terminal.

  1. Go to claude.ai/download in your web browser
  2. Click Download for Mac
  3. Open the downloaded file (it will be in your Downloads folder)
  4. Drag the Claude icon into your Applications folder
  5. Open Claude from your Applications folder (or press Command+Space → type Claude → click it)
  6. Log in with your Claude account (email + password)
Troubleshooting

"Claude can't be opened because it is from an unidentified developer" → Go to System Settings → Privacy & Security → scroll down and click "Open Anyway".

Don't see the download? → Check your Downloads folder. The file is called something like Claude.dmg.

STEP 2: Install Claude Code

The desktop app handles most things, but you also need to install Claude Code for scheduled/automated workflows. This is a one-time setup — just copy-paste one line.

  1. Open Terminal (press Command+Space → type Terminal → click it)
  2. Copy this entire line (triple-click to select, then Command+C):
curl -fsSL https://claude.ai/install.sh | bash
  1. Paste it into Terminal (Command+V) and press Enter
  2. Wait 1-2 minutes. When it's done you'll see Claude Code installed successfully
  3. Close Terminal (Command+Q) and reopen it (Command+Space → Terminal)
Troubleshooting

"Permission denied" → Copy-paste this instead:
sudo curl -fsSL https://claude.ai/install.sh | bash
It will ask for your Mac password. Type it (you won't see the letters — that's normal) and press Enter.

STEP 3: Turn Off Permission Prompts

Do this now! Without this step, Claude Code will ask "Allow this action?" before every single thing it does. It makes Claude Code almost unusable. This takes 30 seconds to fix.

Click-by-click:

  1. Open the Claude desktop app
  2. Click Claude in the top menu bar (next to the Apple logo)
  3. Click Settings...
  4. Click "Claude Code" in the settings sidebar
  5. Find "Allow bypass permissions mode" and turn it ON
  6. Close Settings
  7. Now when you use the Code tab, look for the "Bypass permissions" option next to the send button and select it

That's it. Claude Code will now work smoothly without stopping to ask permission for every action.

What does this do?
It just means Claude won't pause to ask "can I edit this file?" or "can I run this command?" every time. It still follows the instructions you give it — it just doesn't ask for approval on each step. You can always change this back in the same Settings menu.

STEP 4: Set Up Slack Notifications

You do NOT need to create a Slack app or bot. Here are three easy options — pick whichever works for you.

Option A: Email the Slack channel (Easiest — no Slack setup at all)

Every Slack channel has a secret email address. If you already connected Gmail (Step 5), Claude can just email the channel. No Slack app, no webhook, nothing.

One-time setup:

  1. In Slack, open the channel you want to receive messages (e.g., #reports)
  2. Click the channel name at the top to open channel settings
  3. Go to Integrations tab
  4. Click "Send emails to this channel" and copy the email address it gives you
  5. Save that email address somewhere your team can access

Now just tell Claude to email that address:

Send a summary of today's report to our Slack channel by emailing it to: reports-abc123@yourworkspace.slack.com

The email shows up as a message in the Slack channel. That's it.

Why this is great: You already set up Gmail in Step 5. This reuses that connection — no extra setup on the Slack side beyond getting the channel email address. Works for scheduled workflows too.

Option B: Slack MCP Connector (built into Claude Desktop)

Claude Desktop has a built-in Slack connector that handles all the auth for you.

  1. Open the Claude desktop app → Settings
  2. Look for Connectors or MCP Servers
  3. Find Slack and click to enable it
  4. Authorize with your Slack workspace when prompted (just click Allow)
  5. Once connected, just tell Claude "send a message to #reports" in plain English

Option C: GitHub → Slack (for scheduled workflows)

If you're running scheduled workflows through GitHub Actions, GitHub has a native Slack integration. No Claude-side Slack setup needed.

  1. Go to github.com/integrations/slack
  2. Click Add to Slack
  3. Pick your Slack workspace and authorize
  4. In Slack, type: /github subscribe owner/repo workflows
  5. GitHub will now post workflow results directly to your Slack channel
Which should I pick?
  • Option A (email) — best if you don't want to touch Slack settings at all. Just get the channel email and go.
  • Option B (connector) — best if you want Claude to read and search Slack messages too, not just send.
  • Option C (GitHub) — best for scheduled workflow notifications. Set and forget.

STEP 5: Set Up Gmail

Claude can send emails from your own Gmail account. You need to connect the Gmail connector first.

Setup:

  1. Open the Claude desktop app
  2. Go to Settings
  3. Look for Connectors or MCP Servers
  4. Find and enable the Gmail connector
  5. Click Authorize — a Google login page will open in your browser
  6. Log in with your Google account and click Allow on all permissions
  7. Done — Claude can now read and send emails from your Gmail

Once connected, just tell Claude what to email:

Send an email to team@company.com with a summary of today's report. Keep it brief and professional.

The email comes from YOUR Gmail account (your name, your address). It looks like a normal email you sent.

Troubleshooting

"Gmail not connected" → Go back to Settings → Connectors and re-authorize the Gmail connector.

"Permission denied" → When the Google popup appears, make sure you check ALL the permission boxes and click "Allow."

STEP 6: Try Cowork Mode

Cowork is Claude's mode for bigger, multi-step tasks. Instead of just chatting back and forth, Claude plans the work, breaks it into pieces, and executes it — like having an assistant who goes away, does the work, and comes back with finished results.

What Cowork is good for:

  • Building a spreadsheet with formulas and data
  • Creating a presentation from scratch
  • Researching a topic and writing a structured report
  • Organizing files or data across multiple steps
  • Any task that has multiple steps and takes more than a few minutes

How to use it:

  1. Open the Claude desktop app
  2. Click the Cowork tab (at the top, next to the regular Chat and Code tabs)
  3. Describe what you want — be specific about the end result
  4. Claude will create a plan, show it to you, then get to work
  5. When it's done, your files are saved directly to your computer

Example prompts for Cowork:

Create an Excel spreadsheet that tracks our team's weekly KPIs.
Include columns for date, metric name, target, actual, and % achieved.
Add conditional formatting so anything below target shows red.
Save it to my Desktop.
Research the top 5 competitors in [our industry] and create a
comparison table in a Google Doc. Include pricing, key features,
and pros/cons for each.
Best practice: Be specific about what you want the output to look like. Instead of "make a report," say "make a 1-page PDF summary with bullet points and a chart." The more specific you are, the better the result.
Good to know:
- Cowork uses more tokens than regular chat — best for substantial tasks, not quick questions
- Give Cowork all the context it needs each time for best results

STEP 7: Test Everything

Let's make sure it all works.

  1. Open the Claude desktop app
  2. Click the Code tab
  3. Type: What is 2 + 2? — you should get an answer

If you got a response → You're all set!

Troubleshooting

Don't see the Code tab? → Make sure you have the latest version of the Claude desktop app. Close and reopen it.

"Not logged in" errors? → Open Terminal, paste claude auth login, press Enter, and log in through the browser.

Best Practices

1. Be specific.
Bad: "Make a report."
Good: "Create a 1-page summary of this week's sales data with a bar chart comparing regions. Save as PDF to my Desktop."
2. Give Claude context every time.
Tell Claude what files to look at, what format you want, who the audience is. Each conversation starts fresh, so give it everything it needs.
3. Use the right mode for the job.
Quick question or code fix? → Code tab.
Multi-step project with deliverables? → Cowork tab.
4. Use CLAUDE.md for shared team instructions.
If your team works on a shared project/repo, your team lead can create a CLAUDE.md file in the project root. Claude reads this automatically at the start of every session. Great for things like "always use this format" or "never change files in /config." Run /init in the Code tab to auto-generate one.
5. Start fresh when switching tasks.
If you're moving to a totally different task, start a new conversation. Long conversations with mixed topics confuse Claude and waste context space.

FAQ

Q: Do I use the desktop app or Terminal?
A: Use the desktop app for everyday work. The Code tab and Cowork tab have everything you need. You only need Terminal for the initial install and for setting up scheduled workflows.

Q: What's the difference between the Code tab and Cowork?
A: The Code tab is for quick, back-and-forth conversations — like chatting with Claude about code, files, or questions. Cowork is for bigger tasks where Claude plans, executes multiple steps, and delivers finished files.

Q: Will this slow down my computer?
A: No. It only uses resources while you're actively using it.

Q: Does Claude remember my previous conversations?
A: No. Each conversation starts fresh. Always give Claude the context it needs. (Exception: if your project has a CLAUDE.md file, Claude reads that automatically every session.)

Q: How much does it cost?
A: It's included with your Claude Pro/Max subscription. Cowork uses more tokens than regular chat. If your team uses API keys, your team lead manages the budget.
